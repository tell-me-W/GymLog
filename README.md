# GymLog

GymLog는 광고, 계정, 서버 동기화 없이 기기 안에서만 운동을 기록하는 로컬 퍼스트 Android 운동 로그 앱입니다. Kotlin, Jetpack Compose, Room(SQLite)을 사용하며, 네트워크가 없어도 운동 시작, 세트 입력, 기록 조회, 이전 기록 복사가 동작하도록 설계되어 있습니다.

## 핵심 방향

- 모든 운동 데이터는 로컬 Room DB에만 저장합니다.
- 운동 중 입력은 `DRAFT` 세션에 즉시 반영합니다.
- 운동 완료 시 실제 완료 체크된 세트만 `COMPLETED` 기록으로 확정합니다.
- 완료된 과거 운동 기록을 현재 드래프트 운동으로 복사해 다시 수행할 수 있습니다.
- 설정 화면에서 프로필, 앱 버전, 운동 기록 백업/불러오기, 텍스트 기록 추가를 제공합니다.
- 운동은 반복/중량 기준 운동과 시간 기준 유산소 운동을 구분해 입력합니다.
- 자주 묶어 수행하는 종목은 루틴으로 저장해 현재 운동에 한 번에 추가할 수 있습니다.
- 운동 완료 후 요약 텍스트나 인증 이미지를 Android 공유 시트로 공유할 수 있습니다.
- Galaxy/Android 네이티브 사용 경험을 우선합니다.
- 현재 UI는 다크 테마, 하단 바텀시트, 큰 세트 입력 컨트롤을 중심으로 구성되어 있습니다.

## 주요 기능

### 월별 대시보드

첫 화면에서는 월별 운동 캘린더를 확인합니다.

- 선택 월을 `2026년 5월` 형식으로 표시합니다.
- `<`, `>` 버튼으로 이전 달과 다음 달로 이동합니다.
- 완료된 운동이 있는 날짜는 캘린더 셀에 표시됩니다.
- 오늘 날짜는 날짜 아래 작은 점으로 표시됩니다.
- 캘린더 아래에는 선택한 달의 월간 요약을 표시합니다.
- 월간 요약은 `운동 N회`, `총 볼륨 N kg` 형식입니다.
- 월간 요약은 해당 월의 `COMPLETED` 세션과 완료 체크된 세트만 기준으로 계산합니다.
- 진행 중인 드래프트 운동이 있으면 `진행 중 운동 이어하기`로 복귀할 수 있습니다.
- `운동 시작`, `기록 보기` 버튼으로 주요 흐름에 진입합니다.
- 우상단 설정 버튼으로 설정 화면에 진입합니다.

### 설정

설정 화면에서는 사용자 프로필과 운동 기록 관리 기능을 제공합니다.

- 프로필: 키(cm), 몸무게(kg), 성별, 나이를 Room DB에 저장합니다.
- 앱 버전: 현재 앱의 `versionName`과 `versionCode`를 표시합니다.
- `불러오기`: Android Storage Access Framework로 JSON 백업 파일을 선택해 기존 기록에 추가합니다.
- `텍스트로 추가`: 기존 운동 앱에서 복사한 텍스트 기록을 파싱해 완료 기록으로 추가합니다.
- `운동 기록 백업하기`: 완료된 운동 기록을 JSON 파일로 저장합니다.
- 프로필 정보는 운동 기록 백업 JSON에 포함하지 않습니다.
- 백업 불러오기는 기존 기록에 추가하는 방식이며, 시작 시각, 종료 시각, 운동명/세트 구성이 같은 완료 기록은 중복으로 건너뜁니다.

### 텍스트 기록 추가

텍스트 기록 추가는 기존 운동 앱에서 복사한 자유 운동 기록을 여러 세션으로 나누어 저장합니다.

- `[자유 운동] 2026년 5월 15일` 형식의 헤더를 세션 시작으로 인식합니다.
- `#짐워크[자유 운동] 2026년 4월 15일`처럼 헤더 앞에 불필요한 문자열이 붙어도 새 세션으로 인식합니다.
- `28분` 같은 운동 시간은 `endedAt = startedAt + 28분`으로 저장합니다.
- 날짜만 있는 텍스트 기록은 해당 날짜 정오 12:00을 시작 시각으로 저장합니다.
- `45kg x 15회` 형식의 세트를 지원합니다.
- `15회`처럼 중량이 없는 세트는 `0kg x 15회`로 저장합니다.
- `총 볼륨`, `칼로리`, 단독 `#짐워크`, 코드블럭 표시는 저장하지 않고 건너뜁니다.
- 텍스트에 나온 운동명이 기본 운동 라이브러리에 있으면 seed의 타겟 부위와 휴식 시간을 사용합니다.
- DB에 없는 운동명은 `기타` 커스텀 운동으로 자동 생성합니다.
- 파싱된 세트는 모두 완료된 세트로 저장합니다.

### 운동 시작 바텀시트

운동 시작은 하단 바텀시트로 열립니다.

- `빈 운동으로 시작`: 새 `DRAFT` 세션을 생성하고 실시간 로깅 화면으로 이동합니다.
- `이전 날짜 기록 불러오기`: 완료된 과거 기록을 골라 현재 운동으로 복사합니다.
- 기존 드래프트 세션은 새 운동 시작이나 복사 때문에 자동 삭제되지 않습니다. 운동 중 기록 손실을 줄이는 정책입니다.

### 이전 기록 불러오기

완료된 운동 기록을 날짜별로 확인하고 새 운동으로 복사합니다.

- 완료 기록 날짜를 가로 목록에서 선택합니다.
- 선택된 기록의 운동 종목, 세트 구성, 중량, 반복 횟수를 미리 볼 수 있습니다.
- 기록 복사 시 새 `DRAFT` 세션을 생성합니다.
- 복사된 모든 세트의 `isCompleted`는 `false`로 초기화합니다.
- 기록이 없을 때는 빈 운동으로 시작할 수 있습니다.

### 실시간 운동 로깅

운동 중 화면에서는 세션 경과 시간, 휴식 시간, 운동 종목, 세트별 중량/반복 수를 입력합니다.

- 세션 시작 후 경과 타이머가 표시됩니다.
- `운동 완료` 버튼으로 세션을 완료합니다.
- 운동 종목 카드는 운동명, 타겟 부위, 기본 휴식 시간을 보여줍니다.
- 운동 종목 카드에는 휴지통 아이콘 버튼이 있으며, 확인 후 해당 운동과 입력한 세트를 삭제할 수 있습니다.
- 세트 컨트롤은 `-  세트  +` 레이아웃입니다.
- `+`를 누르면 가장 마지막 세트의 중량과 반복 수를 그대로 복사해 새 세트를 추가합니다.
- `-`를 누르면 마지막 세트를 삭제합니다. 1번 세트만 남아 있을 때는 삭제하지 않습니다.
- 마지막 운동 카드에서 세트를 추가하면 화면이 아래로 따라 내려가 세트 추가 버튼 위치를 유지합니다.
- 각 세트는 세트 번호, `kg`, `reps`, 완료 체크 버튼을 가집니다.
- 시간 기준 운동은 `kg`, `reps` 대신 분 단위 시간 입력과 완료 체크 버튼을 보여줍니다.
- 한 세트의 `kg` 또는 `reps`를 수정하면 같은 운동 종목 안의 완료되지 않은 다른 세트에도 같은 값이 반영됩니다.
- 이미 완료 체크된 세트는 자동 값 전파에서 제외됩니다.
- 완료 체크 시 해당 운동의 기본 휴식 시간을 기준으로 휴식 타이머가 시작됩니다.
- 휴식 타이머 알림은 휴식이 끝났을 때만 1회 표시됩니다.

### 운동 종목 추가

운동 종목 추가는 하단 바텀시트에서 처리합니다.

- 여러 운동 종목을 한 번에 선택해 추가할 수 있습니다.
- 운동명 검색을 지원합니다.
- 타겟 부위 필터는 `전체`, `가슴`, `등`, `하체`, `어깨`, `팔`, `복근`, `기타`, `유산소`입니다.
- `전체` 선택 시 모든 운동 종목을 보여줍니다.
- 최근 2개월 안에 수행한 운동은 `최근 yyyy.MM.dd · N세트` 형식으로 표시합니다.
- 타겟 부위 필터 선택 시 최근 수행일이 있는 종목을 최신순으로 먼저 보여주고, 기록이 없는 종목은 뒤쪽에 이름순으로 보여줍니다.
- 선택한 종목들은 우측 상단 `+` 버튼으로 한 번에 추가합니다.
- 커스텀 종목 추가 폼은 기본으로 접혀 있고, 버튼으로 열고 닫을 수 있습니다.
- 커스텀 종목은 이름, 타겟 부위, 기본 휴식 시간을 지정해 추가합니다.
- 커스텀 종목의 타겟 부위에는 `전체`를 저장하지 않고 실제 운동 분류 중 하나를 저장합니다.
- `기타 횟수 기준 운동`은 `기타` 분류의 반복/중량 기준 운동으로 유지합니다.
- `기타 시간 기준 운동`은 `유산소` 분류의 시간 기준 운동으로 제공합니다.
- 기본 운동 라이브러리는 앱 시작 시 DB와 동기화됩니다. 기존에 같은 이름의 기본 종목이 `기타` 커스텀으로 들어간 경우 seed 기준 분류로 보정합니다.

### 루틴

운동 종목 추가 바텀시트에는 `종목` 탭과 `루틴` 탭이 있습니다.

- `종목` 탭에서 여러 운동을 선택합니다.
- `루틴` 탭으로 이동하면 선택한 운동 이름 목록을 확인할 수 있습니다.
- 루틴명을 입력해 선택한 종목 목록을 루틴으로 저장합니다.
- 루틴은 종목 목록과 순서만 저장하며, 세트 수, 중량, 반복 횟수, 시간 값은 저장하지 않습니다.
- 저장된 루틴을 누르면 현재 드래프트 운동에 루틴 종목들이 순서대로 추가됩니다.
- 루틴으로 추가된 각 운동은 기본 1세트로 시작합니다.
- 저장된 루틴은 루틴 탭에서 삭제할 수 있습니다.

### 운동 완료 저장 정책

`운동 완료`를 누르면 저장할 데이터를 정리합니다.

- 완료 체크된 세트만 기록에 저장합니다.
- 완료 체크되지 않은 세트는 삭제합니다.
- 완료 체크된 세트가 하나도 없는 운동 종목은 기록에서 제외합니다.
- 세션 전체에 완료 체크된 세트가 하나도 없으면 완료 기록으로 저장하지 않고 드래프트 세션을 삭제합니다.
- 저장된 기록의 총 볼륨은 반복/중량 기준 세트의 `중량 kg * 반복 횟수` 합계로 계산합니다.
- 시간 기준 운동 세트는 수행 기록으로 저장하지만 총 볼륨에는 포함하지 않습니다.

이 정책 덕분에 운동 기록 화면에는 실제로 수행한 세트만 남습니다.

### 운동 완료 공유

운동 완료 화면에서는 완료한 운동 요약을 공유할 수 있습니다.

- `텍스트 공유`: 날짜, 운동 시간, 총 볼륨, 종목 수, 세트 수를 텍스트로 만들어 Android 공유 시트를 엽니다.
- `이미지 공유`: 앱 내부에서 인증 카드 PNG를 생성해 Android 공유 시트를 엽니다.
- 공유 파일은 앱 cache 디렉터리에 생성되며, 원본 운동 기록은 변경하지 않습니다.

### 운동 기록 보기

`기록 보기` 화면에서는 완료된 운동 목록과 상세 기록을 확인합니다.

- 완료된 운동 목록은 최신순으로 표시합니다.
- `완료한 운동` 헤더는 목록 스크롤과 상관없이 화면에 고정되어 있습니다.
- `대시보드로 돌아가기` 버튼은 운동 기록 타이틀 오른쪽에 있습니다.
- 각 기록은 날짜와 운동 시간을 보여줍니다.
- 기록 삭제 버튼을 제공하며, 삭제 전 확인 다이얼로그를 표시합니다.
- 상세 화면에서는 날짜, 총 볼륨, 운동 시간, 종목 수, 세트 수를 확인합니다.
- 운동별 세트 내역을 `kg x reps` 형태로 볼 수 있습니다.
- 시간 기준 운동 세트는 분 단위 시간으로 볼 수 있습니다.
- 상세 화면에서 해당 기록을 새 드래프트 운동으로 복사해 시작할 수 있습니다.

### 뒤로가기 동작

Android 시스템 뒤로가기는 앱 내부 화면 이동을 우선합니다.

- 기록 상세 화면에서 뒤로가기: 운동 기록 목록으로 이동
- 운동 시작, 이전 기록 불러오기, 운동 기록, 실시간 로깅, 운동 완료 화면에서 뒤로가기: 대시보드로 이동
- 대시보드에서 뒤로가기: 시스템 기본 동작

## 현재 MVP 범위

현재 구현된 MVP 기능은 다음과 같습니다.

- 월별 캘린더 대시보드
- 월 이동
- 월간 운동 횟수/총 볼륨 요약
- 진행 중 드래프트 운동 이어하기
- 로컬 Room DB 저장
- 기본 운동 라이브러리 seed
- 기본 운동 라이브러리 동기화와 잘못된 커스텀 분류 보정
- 운동 종목 검색/필터
- 최근 2개월 수행 기록 표시와 최신순 정렬
- 여러 운동 종목 한 번에 추가
- 커스텀 운동 추가
- 시간 기준 유산소 운동
- 커스텀 루틴 생성/불러오기/삭제
- 설정 화면
- 프로필 저장
- 앱 버전 표시
- JSON 운동 기록 백업
- JSON 운동 기록 불러오기
- 텍스트 운동 기록 여러 세션 추가
- 빈 운동 시작
- 이전 완료 기록 복사
- 실시간 세트 입력
- 세트 추가/삭제 컨트롤
- 마지막 세트 값 복사 추가
- 세트 완료 체크
- 같은 운동 안의 미완료 세트 값 전파

## 데이터 모델

### Exercise

운동 종목입니다.

- `name`: 운동명
- `targetArea`: 타겟 부위
- `isCustom`: 커스텀 종목 여부
- `defaultRestSeconds`: 기본 휴식 시간
- `inputType`: `REPS` 또는 `DURATION`

### WorkoutSession

운동 세션입니다.

- `startedAtMillis`: 시작 시각
- `endedAtMillis`: 완료 시각
- `status`: `DRAFT` 또는 `COMPLETED`

### SessionExercise

세션 안에 포함된 운동 종목입니다.

- `sessionId`: 소속 세션
- `exerciseId`: 운동 종목
- `sortOrder`: 표시 순서

### WorkoutSet

세트 기록입니다.

- `sessionExerciseId`: 소속 세션 운동
- `sortOrder`: 세트 순서
- `weightKg`: 중량
- `reps`: 반복 횟수
- `durationSeconds`: 시간 기준 운동의 수행 시간
- `isCompleted`: 수행 완료 여부

### Routine

사용자가 저장한 커스텀 루틴입니다.

- `name`: 루틴명
- 루틴 운동 목록: 운동 종목 ID와 정렬 순서
- 세트 구성, 중량, 반복 횟수, 시간 값은 저장하지 않습니다.

### UserProfile

설정 화면에서 저장하는 사용자 프로필입니다.

- `heightCm`: 키
- `weightKg`: 몸무게
- `gender`: 성별
- `age`: 나이

## 주요 동작 정책

### Draft-first 저장

운동을 시작하면 즉시 `DRAFT` 세션이 생성됩니다. 세트 추가, 중량 수정, 반복 수 수정, 완료 체크는 드래프트에 저장됩니다. 사용자가 `운동 완료`를 누를 때만 완료 저장 정책을 거쳐 `COMPLETED` 기록이 됩니다.

### 완료 체크 기준 저장

운동 완료 시 `isCompleted = true`인 세트만 저장 대상입니다. 미완료 세트와 빈 운동 종목은 제거됩니다. 완료 세트가 전혀 없으면 해당 세션은 완료 기록으로 남기지 않습니다.

### 이전 기록 복사

복사 가능한 원본은 `COMPLETED` 세션입니다. 복사 결과는 `DRAFT` 세션이고, 모든 세트의 완료 상태는 `false`로 초기화됩니다.

### 세트 값 전파

운동 중 한 세트의 중량 또는 반복 횟수를 수정하면 같은 운동 종목의 미완료 세트에도 동일한 값이 적용됩니다. 이미 완료한 세트와 다른 운동 종목은 변경하지 않습니다.

### 세트 추가/삭제

새 세트는 마지막 세트의 중량과 반복 횟수를 기본값으로 사용합니다. 세트 삭제는 마지막 세트만 대상으로 하며, 한 운동에는 최소 1개 세트를 유지합니다.

시간 기준 운동의 새 세트는 마지막 세트의 시간 값을 기본값으로 사용합니다.

### 운동 입력 타입

반복/중량 기준 운동은 `kg`, `reps`, 완료 체크를 입력합니다. 시간 기준 운동은 분 단위 시간과 완료 체크만 입력합니다. `유산소` 분류의 시간 기준 운동은 볼륨 계산에서 제외됩니다.

### 커스텀 루틴

루틴은 자주 함께 수행하는 종목 목록을 빠르게 추가하기 위한 기능입니다. 루틴에는 운동 종목과 순서만 저장하고, 운동 중 실제 세트 수와 값은 현재 드래프트에서 새로 입력합니다.

### 휴식 타이머 알림

세트를 완료 체크하면 해당 운동의 기본 휴식 시간으로 카운트다운이 시작됩니다. Android 로컬 알림은 휴식이 끝났을 때만 1회 표시합니다.

### 월간 요약

대시보드 월간 요약은 선택한 달의 `COMPLETED` 세션만 세고, 완료 체크된 세트의 `중량 kg * 반복 횟수`만 합산합니다. 드래프트 세션, 다른 달 세션, 미완료 세트는 제외합니다.

### 백업과 불러오기

운동 기록 백업은 완료된 세션, 세션 운동, 세트, 필요한 운동 종목 정보를 JSON으로 저장합니다. 백업 불러오기는 기존 기록을 유지한 채 추가하며, 시작 시각, 종료 시각, 운동명 순서, 세트 순서, 중량, 반복 횟수가 모두 같은 기록은 중복으로 보고 건너뜁니다.

### 기본 운동 분류 보정

앱 시작 시 seed 기본 운동 목록을 DB와 동기화합니다. DB에 같은 이름의 기본 종목이 없으면 추가하고, 같은 이름이 커스텀/`기타` 등 잘못된 분류로 들어간 경우 seed의 타겟 부위, 휴식 시간, 기본 종목 여부로 보정합니다.

## 알림 권한

Android 13 이상에서는 휴식 타이머 완료 알림을 위해 `POST_NOTIFICATIONS` 권한을 요청합니다. 권한이 허용되면 세트 완료 체크 후 휴식 시간이 끝났을 때 로컬 알림이 표시됩니다.

## 설계 원칙

- 운동 중 입력 흐름을 빠르게 유지합니다.
- 네트워크 연결을 전제로 하지 않습니다.
- 실제 수행한 세트만 기록에 남깁니다.
- 이전 기록 복사를 핵심 기능으로 취급합니다.
- 기능이 늘어나도 로컬 데이터 소유권은 사용자 기기에 둡니다.
