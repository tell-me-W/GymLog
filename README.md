# GymLog

GymLog는 광고, 계정, 서버 동기화 없이 기기 안에서만 운동을 기록하는 로컬 퍼스트 Android 운동 로그 앱입니다. Kotlin, Jetpack Compose, Room(SQLite)을 사용하며, 네트워크가 없어도 운동 시작, 세트 입력, 기록 조회, 이전 기록 복사가 동작하도록 설계되어 있습니다.

## 핵심 방향

- 로컬 DB에만 운동 데이터를 저장합니다.
- 운동 중 입력은 Draft 세션에 즉시 반영합니다.
- 운동 완료 시 실제 완료 체크된 세트만 기록으로 확정합니다.
- 완료된 과거 운동 기록을 현재 드래프트 운동으로 복사해 다시 수행할 수 있습니다.
- Galaxy/Android 네이티브 앱을 우선 대상으로 합니다.

## 주요 기능

### 월별 대시보드

앱 첫 화면에서는 월별 운동 캘린더를 볼 수 있습니다.

- 현재 선택된 월을 `2026년 5월` 형식으로 표시합니다.
- `<`, `>` 버튼으로 이전 달과 다음 달을 이동합니다.
- 완료된 운동이 있는 날짜는 캘린더 셀에 하이라이트와 점으로 표시합니다.
- 진행 중인 드래프트 운동이 있으면 `진행 중 운동 이어하기` 버튼으로 복귀할 수 있습니다.
- `운동 시작`, `운동 기록 보기`로 주요 흐름에 진입합니다.

### 운동 시작

운동 시작 화면에서는 두 가지 방식으로 운동을 시작합니다.

- `빈 운동으로 시작`: 새 DRAFT 세션을 생성합니다.
- `이전 날짜 기록 불러오기`: 완료된 과거 운동을 선택해 현재 DRAFT 세션으로 복사합니다.

기존 드래프트 세션은 새 운동 시작이나 복사 때문에 자동 삭제되지 않습니다. 사용 중 기록 손실을 줄이는 쪽으로 정책을 잡았습니다.

### 실시간 운동 로깅

운동 중 화면에서는 세션 경과 시간, 운동 종목, 세트별 중량/반복 수를 입력합니다.

- 세션 시작 시 경과 타이머가 동작합니다.
- 운동 종목을 추가하거나 삭제할 수 있습니다.
- 한 번에 여러 운동 종목을 선택해 추가할 수 있습니다.
- 커스텀 운동 종목을 이름, 타겟 부위, 기본 휴식 시간과 함께 추가할 수 있습니다.
- 각 운동에는 세트를 추가/삭제할 수 있습니다.
- 각 세트는 `kg`, `reps`, 완료 체크박스를 가집니다.
- 한 세트의 `kg` 또는 `reps`를 수정하면 같은 운동 종목 안의 완료되지 않은 다른 세트에도 같은 값이 반영됩니다.
- 완료 체크된 세트는 자동 값 전파에서 제외됩니다.
- 완료 체크 시 해당 운동의 기본 휴식 시간을 기준으로 휴식 타이머와 로컬 알림이 시작됩니다.

### 운동 완료 저장 정책

운동 완료 버튼을 누르면 저장 전 데이터를 정리합니다.

- 완료 체크된 세트만 기록에 저장합니다.
- 완료 체크되지 않은 세트는 삭제합니다.
- 완료 체크된 세트가 하나도 없는 운동 종목은 기록에서 제외합니다.
- 세션 전체에 완료 체크된 세트가 하나도 없으면 완료 기록으로 저장하지 않고 드래프트 세션을 삭제합니다.
- 저장된 기록의 총 볼륨은 `중량 kg * 반복 횟수`의 합계로 계산합니다.

이 정책 덕분에 운동 기록 화면에는 실제로 수행한 세트만 남습니다.

### 이전 기록 복사

완료된 운동 기록은 현재 드래프트 운동으로 복사할 수 있습니다.

- 운동 종목 순서와 세트 구성, 중량, 반복 횟수를 복사합니다.
- 복사된 세트의 완료 상태는 항상 `false`로 초기화합니다.
- 복사 후 사용자는 새 운동처럼 세트를 다시 체크하며 진행할 수 있습니다.

### 운동 기록 조회

`운동 기록 보기` 화면에서는 완료된 운동 목록과 상세 기록을 확인합니다.

- 완료된 운동 목록을 최신순으로 표시합니다.
- 각 기록은 날짜와 운동 시간을 보여줍니다.
- 기록 상세 화면에서는 날짜, 총 볼륨, 운동 시간, 종목 수, 세트 수를 확인합니다.
- 운동별 세트 내역을 `kg x reps` 형태로 볼 수 있습니다.
- 상세 화면에서 `이 기록으로 운동 시작`을 눌러 해당 기록을 새 드래프트 운동으로 복사할 수 있습니다.

### 운동 라이브러리

앱 시작 시 기본 운동 라이브러리를 로컬 DB에 seed 합니다.

- 한국어 운동명 기반 기본 운동 약 30개 이상을 제공합니다.
- 타겟 부위는 `가슴`, `등`, `하체`, `어깨`, `팔`, `복근`으로 구성합니다.
- 타겟 부위 필터로 추가할 운동을 좁혀 볼 수 있습니다.
- 사용자는 커스텀 운동 종목을 추가할 수 있습니다.

## 현재 MVP 범위

현재 구현된 MVP 범위는 다음과 같습니다.

- 월별 캘린더와 월 이동
- 로컬 Room DB 저장
- 기본 운동 라이브러리 seed
- 커스텀 운동 추가
- 빈 운동 시작
- 이전 완료 기록 복사

- 운동 기록 목록/상세 조회

- 계정 기능

## 기술 스택

- Language: Kotlin
- UI: Jetpack Compose, Material 3
- Architecture: MVVM + Repository
- Database: Room(SQLite)
- Async: Kotlin Coroutines, Flow
- Build: Gradle, Android Gradle Plugin, KSP
- Test: JUnit, kotlinx-coroutines-test, Room testing for instrumented tests

## 데이터 모델

### Exercise

운동 종목입니다.

- `name`: 운동명
- `targetArea`: 타겟 부위
- `isCustom`: 커스텀 종목 여부
- `defaultRestSeconds`: 기본 휴식 시간

### WorkoutSession

운동 세션입니다.

- `startedAtMillis`: 시작 시각
- `endedAtMillis`: 완료 시각
- `status`: `DRAFT` 또는 `COMPLETED`

### SessionExercise

세션 안에 포함된 운동 종목입니다.

- `sessionId`: 소속 세션
- `exerciseId`: 운동 종목
- `sortOrder`: 표시 순서

### WorkoutSet

세트 기록입니다.

- `sessionExerciseId`: 소속 세션 운동
- `sortOrder`: 세트 순서
- `weightKg`: 중량
- `reps`: 반복 횟수
- `isCompleted`: 수행 완료 여부

## 주요 동작 정책

### Draft-first 저장

운동을 시작하면 즉시 DRAFT 세션이 생성됩니다. 세트 추가, 중량 수정, 반복 수 수정, 완료 체크는 드래프트에 저장됩니다. 사용자가 운동 완료를 누를 때만 COMPLETED 기록이 됩니다.

### 완료 체크 기준 저장

운동 완료 시 `isCompleted = true`인 세트만 저장 대상입니다. 미완료 세트와 빈 운동 종목은 제거됩니다. 완료 세트가 전혀 없으면 해당 세션은 완료 기록으로 남기지 않습니다.

### 이전 기록 복사

복사 가능한 원본은 COMPLETED 세션입니다. 복사 결과는 DRAFT 세션이고, 모든 세트의 완료 상태는 false로 초기화됩니다.

### 세트 값 전파

운동 중 한 세트의 중량 또는 반복 횟수를 수정하면 같은 운동 종목의 미완료 세트에도 동일한 값이 적용됩니다. 이미 완료한 세트와 다른 운동 종목은 변경하지 않습니다.

## 개발 환경

권장 환경은 다음과 같습니다.

- JDK 17
- Android Studio
- Android SDK 35
- Windows 환경에서는 예시처럼 JDK 경로를 지정해 실행할 수 있습니다.

```powershell
$env:JAVA_HOME='C:\Program Files\Java\jdk-17'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
```

## 빌드 및 테스트

전체 JVM 테스트를 실행합니다.

```powershell
.\gradlew.bat test
```

디버그 APK를 빌드합니다.

```powershell
.\gradlew.bat assembleDebug
```

현재 확인된 검증 기준은 다음과 같습니다.

- `.\gradlew.bat test`
- `.\gradlew.bat assembleDebug`

## 테스트 범위

현재 테스트는 핵심 도메인/Repository 동작을 중심으로 구성되어 있습니다.

- 총 볼륨 계산
- 세션 요약 계산
- 월 표시 포맷
- 드래프트 세션 보존
- 완료 세트만 저장
- 완료 세트가 없는 세션 미저장
- 이전 기록 복사 시 완료 상태 초기화
- 같은 운동의 미완료 세트 값 전파

Instrumented test에는 Room 기반 Repository 동작 검증이 포함되어 있습니다.

## 알림 권한

Android 13 이상에서는 휴식 타이머 알림을 위해 `POST_NOTIFICATIONS` 권한을 요청합니다. 권한이 허용되면 세트 완료 체크 시 휴식 타이머 진행/완료 알림이 표시됩니다.

## 설계 원칙

- 운동 중 입력 흐름을 빠르게 유지합니다.
- 네트워크 연결을 전제로 하지 않습니다.
- 실제 수행한 세트만 기록에 남깁니다.
- 이전 기록 복사를 핵심 기능으로 취급합니다.
- 기능이 늘어나도 로컬 데이터 소유권을 사용자 기기에 둡니다.
