# GymLog UI Guide

`ui` 영역은 ViewModel, Compose 화면 보조 로직, 공유 콘텐츠, 휴식 타이머 UI 정책을 담당합니다. 저장 정책은 data/domain 계층에 두고 UI는 사용자 흐름과 상태 연결에 집중합니다.

## Responsibilities

- `GymLogViewModel.kt`: repository 주입, StateFlow, 화면 이벤트 메서드.
- UI helper files: 화면 상태, 표시 포맷, 정렬, 메시지, 공유 텍스트/이미지 정책.
- `rest`: 휴식 타이머와 로컬 알림 정책.
- `MainActivity.kt`: Activity entry, 최상위 routing, Compose wiring.

## UI Rules

- 첫 화면은 대시보드입니다.
- 대시보드는 월별 캘린더, 선택 월, 이전/다음 월 이동, 월간 요약, 운동 시작, 운동 기록 진입, 설정 진입을 제공합니다.
- 월간 요약은 캘린더 아래에 둡니다.
- 설정 화면은 프로필, 앱 버전, 운동 기록 불러오기, 텍스트로 추가, 운동 기록 백업하기를 제공합니다.
- 운동 종목 추가 화면은 여러 종목을 한 번에 선택할 수 있어야 합니다.
- 운동 종목 필터의 `전체`는 필터 없음 상태입니다.
- 운동 종목 필터에는 `기타`도 포함합니다.
- 운동 종목 목록은 최근 2개월 수행 기록이 있는 종목을 최신순으로 우선 정렬하고, 기록이 없는 종목은 이름순으로 정렬합니다.
- 최근 수행 기록은 `최근 yyyy.MM.dd · N세트` 형식으로 표시합니다.
- 커스텀 종목 추가 폼은 기본으로 접고 사용자가 열고 닫을 수 있게 합니다.
- 커스텀 종목의 분류에는 `전체`를 저장하지 않습니다.
- 운동 기록 목록에는 기록 삭제 기능과 삭제 확인을 제공합니다.
- Android 뒤로가기는 앱 내부 화면 이동을 우선하고, 대시보드에서는 시스템 기본 동작을 허용합니다.
- 텍스트는 한국어 UI 문구를 기본으로 합니다.

## ViewModel And Helper Rules

- ViewModel에는 화면 이벤트와 StateFlow 조립을 남기고, 순수 로직은 helper/domain으로 분리합니다.
- one-shot UI 이벤트는 상태에 섞지 말고 Flow 등 명확한 이벤트 채널을 사용합니다.
- 날짜/시간 포맷, summary mapping, 화면 back 정책, 메시지 문구는 작은 테스트 가능한 helper로 둡니다.
- 플랫폼 권한, 알림, 타이머 정책은 가능한 부분을 순수 함수로 분리합니다.

## Testing Focus

- 화면별 뒤로가기 목적지.
- 날짜/시간 표시 포맷.
- 월간 요약 표시 상태 mapping.
- 최근 기록 기반 운동 종목 정렬.
- 루틴 생성 가능 조건과 선택 종목 표시.
- 휴식 타이머 알림 정책.
- 공유 텍스트 생성과 사용자 메시지 문구.
