# Backup And Import

백업과 가져오기는 완료 운동 기록만 대상으로 하며, 기존 기록을 보존하면서 중복을 건너뜁니다.

## JSON Backup

- JSON 백업은 완료 운동 기록만 포함합니다.
- 사용자 프로필은 백업 JSON에 포함하지 않습니다.
- 백업에는 완료된 세션, 세션 운동, 세트, 필요한 운동 종목 정보를 저장합니다.
- 백업 파일 생성은 원본 운동 기록을 변경하지 않습니다.

## JSON Import

- 불러오기는 Android Storage Access Framework로 JSON 백업 파일을 선택합니다.
- 가져온 기록은 기존 기록에 추가합니다.
- 시작 시각, 종료 시각, 운동명 순서, 세트 순서, 중량, 반복 횟수가 모두 같은 완료 기록은 중복으로 보고 건너뜁니다.
- 가져오기 중 기본 운동명은 seed의 타겟 부위와 휴식 시간을 우선 사용합니다.
- DB에 없는 운동명은 `기타` 커스텀 운동으로 자동 생성합니다.

## Text Import

- 텍스트 기록 추가는 기존 운동 앱에서 복사한 자유 운동 기록을 여러 세션으로 나누어 저장합니다.
- `[자유 운동] yyyy년 M월 d일` 헤더마다 세션을 분리합니다.
- `#짐워크[자유 운동] ...`처럼 헤더 앞에 불필요한 문자열이 붙어도 세션 헤더로 처리합니다.
- 날짜만 있는 텍스트 기록은 해당 날짜 정오 12:00을 시작 시각으로 저장합니다.
- `28분` 같은 운동 시간은 `endedAt = startedAt + 28분`으로 저장합니다.
- `45kg x 15회` 형식의 세트를 지원합니다.
- 중량 없는 `15회` 세트는 `0kg x 15회`로 저장합니다.
- `총 볼륨`, `칼로리`, 단독 `#짐워크`, 코드블럭 표시는 저장하지 않고 건너뜁니다.
- 파싱된 세트는 모두 완료된 세트로 저장합니다.
