# Product Rules

GymLog의 제품 정책은 로컬 저장, 빠른 운동 입력, 실제 수행 세트만 기록하는 흐름을 중심으로 합니다.

## Product Basics

- Galaxy/Android 전용 Kotlin 네이티브 앱입니다.
- 서버, 계정, 동기화, 광고, 결제 유도는 MVP 범위에 넣지 않습니다.
- 모든 운동 데이터는 로컬 Room DB에만 저장합니다.
- 첫 화면은 대시보드입니다.
- 한국어 UI 문구를 기본으로 합니다.

## Core Behavior

- 운동 시작 즉시 `DRAFT` 세션을 생성합니다.
- 운동 중 입력은 드래프트에 즉시 저장합니다.
- `운동 완료` 시 완료 체크된 세트만 `COMPLETED` 기록에 남깁니다.
- 완료 체크되지 않은 세트는 완료 기록 저장 전에 제거합니다.
- 완료 세트가 없는 운동 종목은 완료 기록에서 제외합니다.
- 완료 세트가 하나도 없는 세션은 완료 기록으로 저장하지 않고 드래프트를 삭제합니다.
- 캘린더와 운동 기록 화면에는 `COMPLETED` 세션만 표시합니다.
- 대시보드 월간 요약은 선택 월의 `COMPLETED` 세션만 세고, 완료 세트의 `중량 * 반복 횟수`만 합산합니다.

## Dashboard And History

- 대시보드는 월별 캘린더, 선택 월, 이전/다음 월 이동, 월간 요약, 운동 시작, 운동 기록 진입, 설정 진입을 제공합니다.
- 월간 요약은 캘린더 아래에 둡니다.
- 운동 기록 목록에는 완료 기록만 최신순으로 표시합니다.
- 기록 삭제 전에는 확인을 받습니다.
- 상세 기록에서는 날짜, 총 볼륨, 운동 시간, 종목 수, 세트 수, 운동별 세트 내역을 표시합니다.

## Navigation

- Android 뒤로가기는 앱 내부 화면 이동을 우선합니다.
- 기록 상세 화면에서는 운동 기록 목록으로 돌아갑니다.
- 운동 시작, 이전 기록 불러오기, 운동 기록, 실시간 로깅, 운동 완료 화면에서는 대시보드로 돌아갑니다.
- 대시보드에서는 시스템 기본 뒤로가기 동작을 허용합니다.
